Was ist GMDB?
GMDB ist die Abkürzung für GermanMangosDataBase!
GMDB ist das derzeit größte deutschsprachige Projekt, dass sich zum Ziel gesetzt hat Datenbanken von den 2 Projekten (MaNGOS, Trinity) auf Deutsch zu übersetzen!
GMDB teilt sich in 2 Unterprojekte auf:
  1. GMDB GermanMangosDataBase
    Dieses Projekt kümmert sich um die Datenbanken von MaNGOS (z.B. UDB (UDB on Sourceforge))
  2. GTDB GermanTrinityDataBase
    Dieses Projekt kümmert sich um die Datenbanken von Trinity (z.B. TrinityDB (TrinityDB SVN))
Die 2 Datenbanken die hier aufgelistet sind, sind nur Beispiele!
Alle Dateien sind UPDATE-Files!
Das heißt, ihr könnt sie auch für andere DBs verwenden!

Wie verwende ich die GMDB?
  1. Zuerst lädtst du dir die neuste GMDB herunter!
    Du findest das letzte offizielle Release im Downloadbereich!
    Aktuellere Releases gibt es übers SVN! (Wie du diese bekommst wird in einem anderen Teil des FAQ's erklärt!)
  2. Nun entpackst du die soeben erhaltene *.rar Datei (Du benötigt hierzu ein Programm wie WinRar -> Google)
  3. Die entpackte *.sql Datei patched du nun über deine UDB (oder jede andere DB die mit MaNGOS kompatibel ist)
    Wie du patched erfährst du in der weiteren FAQ!

Wie lese ich die AKTUELLSTEN GMDB Dateien aus dem SVN aus?
  1. Lade dir zuerst TortoiseSVN aus dem Netz! Du findest es HIER
  2. Installiere das Programm!
  3. Nun erstellst du einen Ordner mit dem Namen GMDB (kann auch anders heißen)
  4. Rechtsklick auf den neuen Ordner und wähle "SVN Auschecken"
  5. Als "URL des Projectarchivs" gibst du folgenden Link ein: https://gm-db.svn.sourceforge.net/svnroot/gm-db
  6. Bestätige nun noch! Darauf wird das komplette SVN ausgelesen! Wenn dies beendet wurde wird dir angezeigt welches Release nun im Ordner ist
  7. Wenn du nun erneut auf den Ordner rechtsklicktst und auf SVN aktualisieren gehst, wird überprüft, ob ein neueres Release vorhanden ist und dies wird dann auch gleich gedownloaded!

Bosse und Wachen in den Städten sind noch Englisch! Was kann ich tun?
Bosse und sprechende NPCs werden von ScriptDev2 festgelegt!
SD2 hat erst vor kurzem Multilanguagesupport eingeführt!
D.h. es sind noch nicht alle Texte in die SD2-Datenbank übertragen!
Die bereits übertragenen Texte (und übersetzten) findest du im SVN im entsprechenden ScriptDev2 Ordner! Diese Dateien musst du in deine SD2-Datenbank patchen!

Wie patche ich bei SQLyog über die Konsole?
  1. Öffne SQLyog und connecte zu deiner DB!
  2. Wähle die zu patchende DB links aus
  3. Öffne das *.sql-File : File -> Open
  4. Nun steht im "Query" alles was in der *.sql Datei enthalten war
  5. Nun noch auf Edit -> Execute Query -> Execute All Queries
  6. SQLyog beginnt zu arbeiten: Fehler werden ignoriert (falls überhaupt vorhanden)! Wenn der Vorgang zu Ende ist erscheint unter "Query" ein Stausbericht!

Wie patche ich mit Navicat?
  1. Öffne Navicat und connecte zu deiner DB!
  2. Rechtsklick auf die Datenbank auf die gepatcht werden soll (Standart: mangos)
  3. Wähle: "Batchdatei ausführen..."
  4. Wähle nun das (bzw. die) SQL-File(s) aus
  5. Öffnen
  6. Navicat beginnt das Patch-File in die Datenbank zu packen
INFO: Sollten Fehler auftreten, versuche zuerst über die Navicat Konsole zu patchen (siehe weitere FAQ)!
Tritt der Fehler danach immernoch auf wende dich bitte an uns (e-Mail an Mordor oder im Sourceforge-Forum oder im deutschen Unterforum im UDB Forum oder in unserem eigene GMDB-Forum)

Wie patche ich mit Navicat über die Konsole?
  1. Öffne Navicat!
  2. Öffne deine Datenbank und Rechtsklick auf die mangos DB (Standartname: mangos)!
  3. Wähle "Konsole..."
  4. Nun auf Datei ---> Laden! Und öffne die *.sql-Datei
  5. Nun noch ENTER
  6. Navicat beginnt zu arbeiten
  7. Wenn keine weiteren Nachrichten in der Konsole erscheinen ist der Patchvorgang abgeschlossen!

Wie patche ich aus der mySQL Konsole unter Windows?
  1. Man öffnet die mySQL Konsole unter Windows unter Start -> Programme -> mySQL -> "MySQL Command Line Client"
  2. Dort wird man nach einem Passwort gefragt -> Passwort eingeben
  3. Wähle nun die Datenbank aus auf die du patchen willst!
    Dies geht mit dem Befehl: Use "DATENBANKNAME" (Standart: mangos)
  4. Nun benötigt ihr den Pfad zu euerem File das ihr patchen wollt!
    Nehmen wir an der file heißt "GMDB.sql" und liegt direkt unter c:/
    Hier würdet ihr folgenden commend benötigen: \. c:/GMDB.sql
  5. Die Konsole arbeitet! Anschließend ist das Patchen beendet!

Wie patche ich unter GNU/Linux?
  1. Man entpackt die *.rar Datei mit dem Befehl unrar x *.rar /path/to/extract
  2. Die entpackte Datei findet ihr unter /path/to/extract
  3. Nun wechseln wir in die mySQL Konsole mit dem Befehl mysql -u [USER] -p und gebt dann euer Passwort ein.
  4. Das weitere Vorgehen ist genau das selbe wie unter Windows und wird unter Wie patche ich aus der mySQL Konsole unter Windows? beschrieben!
Hinweis: Es ist auch möglich freie Programme aus der OpenSource Welt für GNU/Linux zu benutzen die sich auch finden lassen ;)
